สมมติว่าคุณต้องการสร้างรายงานโอกาสการขายใหม่จาก Salesforce ใน Google Sheets  โดยอัตโนมัติทุกครั้งที่มีการสร้างโอกาสการขายใหม่.

**ขั้นตอนที่ 1: เชื่อมต่อ Salesforce กับ Make.com**

1. เข้าสู่ระบบ Make.com <a href="https://www.make.com/en/register?pc=yodome" rel="noindex nofollow">เริ่มต้นการทดลองใช้ Make.com ฟรี</a> และเปิด Scenario ที่คุณต้องการใช้.

2. เพิ่ม Module **"Salesforce > Watch Records"**. Module นี้จะตรวจสอบการเปลี่ยนแปลงใน Salesforce  เช่น  การสร้างโอกาสการขายใหม่  การอัปเดตข้อมูลลูกค้า.

3. คลิก "Add" ในส่วน **Connection** เพื่อเชื่อมต่อกับบัญชี Salesforce ของคุณ.

4. Make.com จะเปิดหน้าต่างใหม่  ให้คุณลงชื่อเข้าใช้บัญชี Salesforce  และอนุญาตให้ Make.com เข้าถึงข้อมูล.

    **(Placeholder for a screenshot showing the Salesforce authorization window in Make.com)**

**ขั้นตอนที่ 2:  กำหนดค่า Module "Salesforce > Watch Records"**

1. **Connection:** เลือก Connection ที่คุณสร้างไว้ในขั้นตอนที่ 1.

2. **Object:** เลือก "Opportunity" (โอกาสการขาย).

3. **Trigger:** เลือก "New Record" (บันทึกใหม่) เพื่อให้ Make.com ทำงานเมื่อมีการสร้างโอกาสการขายใหม่.

4. **Limit (Optional):**  กำหนดจำนวนโอกาสการขายสูงสุดที่ต้องการดึงในแต่ละครั้ง.

    **(Placeholder for a screenshot showing the configuration options for the "Salesforce > Watch Records" module)**

**ขั้นตอนที่ 3: เพิ่มข้อมูลลงใน Google Sheets**

1. เพิ่ม Module **"Google Sheets > Add a Row"** หลัง Module "Salesforce > Watch Records".

2. **กำหนดค่า Module "Add a Row"**:

    * **Connection:** เลือก Connection กับ Google Sheets ของคุณ.
    * **Spreadsheet:**  เลือก Google Sheet ที่คุณต้องการบันทึกข้อมูล เช่น "รายงานโอกาสการขาย".
    * **Worksheet:** เลือก Worksheet ที่ต้องการ.
    * **Values:** แมปข้อมูลจาก Salesforce ไปยังคอลัมน์ใน Google Sheets.  เช่น:

        * "Opportunity Name" แมปกับ `opportunity.name`
        * "Amount" แมปกับ `opportunity.amount` (ใช้ Formatter เพื่อแปลงเป็นสกุลเงิน THB)
        * "Close Date" แมปกับ `opportunity.close_date` (ใช้ Formatter เพื่อจัดรูปแบบวันที่)
        * "Stage" แมปกับ `opportunity.stage_name`
        * "Account Name" แมปกับ  `opportunity.account.name`

## การตรวจสอบข้อผิดพลาด (Error Handling) ใน Make.com สำหรับรายงาน Google Sheets ไดนามิก

ไม่มีระบบใดที่สมบูรณ์แบบ  และบางครั้ง Scenario ของคุณใน Make.com <a href="https://www.make.com/en/register?pc=yodome" rel="noindex nofollow">เริ่มต้นการทดลองใช้ Make.com ฟรี</a> อาจพบข้อผิดพลาดได้  เช่น การเชื่อมต่อกับ Google Sheets ขัดข้อง  ข้อมูลไม่ถูกต้อง  หรือ API ของบริการอื่นๆ มีปัญหา  การตรวจสอบและจัดการข้อผิดพลาดอย่างมีประสิทธิภาพ  เป็นสิ่งสำคัญในการสร้างรายงานไดนามิกที่เชื่อถือได้  และป้องกันการสูญหายของข้อมูล

ในส่วนนี้  เราจะมาดูวิธีการตรวจสอบและจัดการข้อผิดพลาดใน Make.com  เพื่อให้ Scenario ของคุณทำงานได้อย่างราบรื่น  และสร้างรายงาน Google Sheets ไดนามิกได้อย่างมีประสิทธิภาพ.

### 1. ตรวจสอบสถานะการทำงาน (Bubble Execution Status)

หลังจาก Run Scenario  Make.com จะแสดงสถานะการทำงานของแต่ละ Module ด้วยสีต่างๆ:

* **สีเขียว:**  หมายถึงสำเร็จ.
* **สีเหลือง:** หมายถึงมีการแจ้งเตือน  แต่ไม่ใช่ข้อผิดพลาดร้ายแรง.  อาจเป็นการแจ้งเตือนว่าข้อมูลบางส่วนไม่ถูกต้อง  หรือไม่สมบูรณ์.
* **สีแดง:** หมายถึงเกิดข้อผิดพลาด.  Scenario จะหยุดทำงานที่ Module นี้.

**(Placeholder for a screenshot showing a scenario execution in Make.com with green, yellow, and red bubbles)**

คลิกที่ Module สีแดง  หรือสีเหลือง  เพื่อดูรายละเอียดของข้อผิดพลาดหรือการแจ้งเตือน.  ข้อมูลเหล่านี้จะช่วยให้คุณระบุสาเหตุของปัญหา  และแก้ไขได้อย่างตรงจุด.

**ตัวอย่าง:**  หาก Module "Google Sheets > Add a Row"  แสดงสีแดง  อาจเป็นเพราะชื่อคอลัมน์ใน Google Sheets ไม่ตรงกับข้อมูลที่ส่งมาจาก Make.com  หรือบัญชี Google ที่เชื่อมต่อกับ Make.com  ไม่มีสิทธิ์ในการแก้ไข Google Sheet นั้น.
